Protocolos de Red: TCP/IP y OSI

TCP/IP (EN USO)

Internet Protocol (IP): Este protocolo se encarga de direccionar y enrutar los datos a través de la red. Utiliza direcciones IP únicas para identificar cada dispositivo en la red y divide los datos en paquetes que viajan por diversas rutas hasta llegar a su destino.

Transmission Control Protocol (TCP): TCP asegura la entrega fiable de los datos. Antes de transmitir los datos, establece una conexión entre el emisor y el receptor, controla el flujo de información y verifica que los paquetes lleguen en el orden correcto. Si algún paquete se pierde o se corrompe, TCP solicita su retransmisión para garantizar la integridad de la comunicación.

Proceso de Comunicación TCP/IP:

  • Físico: Controla los dispositivos físicos de una red.
  • Enlace: Acceso a la red.
  • Red: Determina la mejor ruta para los datos.
  • Transporte: Admite la comunicación entre los nodos de la red.
  • Aplicación: Apps que se utilizan para usar servicios de la red.

Características Principales:

  • Interoperabilidad: Facilita la comunicación entre diferentes redes y dispositivos.
  • Fiabilidad: Asegura la correcta entrega y orden de los datos.
  • Escalabilidad: Soporta un gran número de dispositivos y redes.

Resumen: TCP/IP es crucial para el funcionamiento de Internet, ya que permite la transmisión eficiente y segura de datos entre dispositivos en una red global.

OSI (EN DESUSO)

Descripción: El modelo OSI (Open Systems Interconnection) es un marco conceptual que estandariza las funciones de comunicación en redes de computadoras en siete capas distintas. Facilita la interoperabilidad entre diferentes sistemas y tecnologías de red al definir un conjunto de protocolos y servicios en cada capa.

Las Siete Capas del Modelo OSI:

  • Capa Física: Transmite los datos a través del medio físico (cables, señales).
  • Capa de Enlace de Datos: Asegura una comunicación sin errores entre dispositivos en una red local.
  • Capa de Red: Dirige y enruta los datos entre diferentes redes usando direcciones IP.
  • Capa de Transporte: Garantiza la entrega correcta de datos entre sistemas finales y maneja el control de flujo y errores.
  • Capa de Sesión: Establece, mantiene y termina sesiones de comunicación entre aplicaciones.
  • Capa de Presentación: Traduce y formatea los datos para que sean entendidos por las aplicaciones, manejando codificación y cifrado.
  • Capa de Aplicación: Proporciona servicios de red directamente a las aplicaciones de usuario (correo electrónico, navegación web).

Características Principales del Modelo OSI:

  • Modularidad: Permite el diseño y la implementación de redes en capas separadas, facilitando la integración y la actualización de componentes.
  • Interoperabilidad: Establece estándares para la comunicación entre diferentes sistemas y tecnologías.
  • Abstracción: Proporciona una separación clara entre las funciones de red, simplificando el diagnóstico y la resolución de problemas.

Resumen: Aunque el modelo OSI es principalmente un concepto de referencia hoy en día, sigue siendo útil para comprender y diseñar la arquitectura de redes, además de ser fundamental en la formación académica sobre redes de comunicación.
